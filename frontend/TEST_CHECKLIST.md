# Чеклист для тестирования панели администратора

## Подготовка к тестированию

- [ ] Backend запущен на `http://localhost:8000`
- [ ] Frontend запущен на `http://localhost:5173`
- [ ] Войти как администратор (демо-аккаунт: `+77770000001` / `admin123`)
- [ ] Перейти на страницу `/admin/dashboard`

---

## 1. Секция "Обзор" (Overview)

### 1.1 Отображение статистики
- [ ] Отображается карточка "Всего студентов" с корректным значением
- [ ] Отображается карточка "Активных курсов" с корректным значением
- [ ] Отображается карточка "Тестов сегодня" с корректным значением
- [ ] Отображается карточка "Выдано сертификатов" с корректным значением
- [ ] Все метрики загружаются из API
- [ ] При загрузке отображается индикатор загрузки (spinner)
- [ ] При ошибке загрузки отображается сообщение об ошибке

### 1.2 Последняя активность
- [ ] Отображается секция "Последняя активность"
- [ ] (Если реализовано) Отображаются последние действия пользователей

### 1.3 Быстрые действия
- [ ] Кнопка "Создать курс" открывает редактор курсов
- [ ] Кнопка "Добавить студента" открывает редактор пользователей
- [ ] Кнопка "Создать тест" открывает редактор тестов

## 2. Секция "Курсы" (Courses)

### 2.1 Отображение списка курсов
- [ ] Отображается список всех курсов из БД
- [ ] Для каждого курса отображается:
  - [ ] Название курса
  - [ ] Категория (с правильным переводом)
  - [ ] Количество студентов
  - [ ] Статус (с правильным бейджем)
- [ ] При загрузке отображается индикатор загрузки
- [ ] При пустом списке отображается сообщение "Нет курсов"

### 2.2 Поиск курсов
- [ ] Поле поиска работает корректно
- [ ] Поиск фильтрует курсы по названию
- [ ] При отсутствии результатов отображается "Курсы не найдены"

### 2.3 Создание курса
- [ ] Кнопка "Создать курс" открывает модальное окно `CourseEditor`
- [ ] Все обязательные поля доступны для заполнения:
  - [ ] Название курса
  - [ ] Описание
  - [ ] Категория (выпадающий список)
  - [ ] Длительность
  - [ ] Статус
- [ ] Можно добавить модули к курсу
- [ ] Можно добавить уроки к модулям
- [ ] Можно привязать тест к курсу (если реализовано)
- [ ] Кнопка "Сохранить" создает курс через API
- [ ] После создания отображается toast-уведомление "Курс успешно создан"
- [ ] Список курсов автоматически обновляется
- [ ] Модальное окно закрывается после успешного сохранения
- [ ] Валидация обязательных полей работает (нельзя сохранить без названия)

### 2.4 Редактирование курса
- [ ] Кнопка "Редактировать" открывает `CourseEditor` с данными курса
- [ ] Все поля предзаполнены корректными значениями
- [ ] Можно изменить название, описание, категорию
- [ ] Можно редактировать модули и уроки
- [ ] Кнопка "Сохранить" обновляет курс через API
- [ ] После обновления отображается toast "Курс успешно обновлен"
- [ ] Список курсов автоматически обновляется
- [ ] Изменения сохраняются в БД

### 2.5 Удаление курса
- [ ] Кнопка "Удалить" показывает диалог подтверждения
- [ ] Диалог содержит название удаляемого курса
- [ ] При подтверждении курс удаляется через API
- [ ] После удаления отображается toast "Курс успешно удален"
- [ ] Список курсов автоматически обновляется
- [ ] Курс исчезает из списка
- [ ] При ошибке удаления отображается сообщение об ошибке

### 2.6 Просмотр студентов курса
- [ ] Кнопка "Студенты" открывает модальное окно `CourseStudentsModal`
- [ ] Отображается список всех студентов на курсе
- [ ] Отображается статистика:
  - [ ] Всего студентов
  - [ ] Завершили курс
  - [ ] В процессе
  - [ ] Средний балл
- [ ] Для каждого студента отображается:
  - [ ] ФИО
  - [ ] Email и телефон
  - [ ] Компания
  - [ ] Прогресс (процент и прогресс-бар)
  - [ ] Оценка
  - [ ] Статус
- [ ] Поиск студентов работает корректно
- [ ] Кнопка "Добавить студентов" открывает `AddStudentsToCourseModal`

### 2.7 Добавление студентов на курс
- [ ] Модальное окно `AddStudentsToCourseModal` открывается корректно
- [ ] Отображается список доступных студентов (не зачисленных на курс)
- [ ] Можно выбрать несколько студентов
- [ ] Поиск студентов работает
- [ ] Фильтрация по роли работает (если реализована)
- [ ] Кнопка "Добавить" добавляет студентов через API
- [ ] После добавления список студентов обновляется
- [ ] Модальное окно закрывается
- [ ] Добавленные студенты появляются в списке студентов курса
